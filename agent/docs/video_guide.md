## 开场白（建议 1 分钟）

大家好，今天给大家介绍一个我独立开发的 AI 反诈系统——**SmartGuard 全民反诈智能助手**。

在电信网络诈骗日益猖獗的今天，传统的人工客服和固定规则系统已经难以应对层出不穷的诈骗手法。我开发这套系统的核心理念是：**让 AI 拥有"感知-决策-干预-进化"四个闭环能力**，像一个永不疲倦的反诈专家，7x24 小时守护用户。

---

## 第一部分：项目背景与核心创新（建议 3 分钟）

### 1.1 问题背景

先看一组数据：近年来电信网络诈骗案件持续高发，诈骗手法不断升级，从传统的电话诈骗，到 AI 语音合成、深度伪造（Deepfake），诈骗分子也在"技术升级"。而传统的反诈手段面临三个核心挑战：

1. **响应滞后** —— 新诈骗手法出现后，规则库更新需要人工维护，存在时间差
2. **维度单一** —— 大多数系统只分析文本，无法处理语音、图片等多模态信息
3. **缺乏进化** —— 无法从真实案例中自动学习新特征

### 1.2 核心创新点

SmartGuard 正是针对这三点做出了技术创新：

| 创新点 | 描述 |
|--------|------|
| **多模态感知** | 同步处理文本、语音、图像、视频，是业界少数能做到音频+图像联合分析的反诈系统 |
| **四闭环架构** | 感知→决策→干预→进化，每个环节无缝衔接 |
| **自适应进化** | 每周自动从公安部、12321、CCTV 等权威数据源学习最新诈骗案例，无需人工维护 |
| **声纹级辅助** | 通过 SenseVoice 提取声学特征，识别语音中的紧张、恐惧等异常情绪信号 |
| **分级响应机制** | 5级风险响应，从温和提醒到监护人强制通知，精准匹配不同风险等级 |

## 第二部分：系统架构（建议 3 分钟）

SmartGuard 采用**前后端分离**架构，后端基于 FastAPI 构建高性能异步 API 服务，前端提供 PC Web 和手机 H5 双端界面。

### 2.1 技术栈一览

```
前端：Vue.js 3 + Element Plus（PC端）/ 移动端H5
后端：Python 3.11 + FastAPI（异步架构）
大模型：通义千问（Qwen-Turbo/Qwen-Max）
向量数据库：ChromaDB（RAG 知识检索）
关系型数据库：SQLite
语音模型：SenseVoiceSmall（阿里达摩院）
图像模型：Qwen-VL-Max + YOLO
OCR：PaddleOCR
```

### 2.2 系统架构图

```
                        ┌──────────────┐
                        │   用户端      │
                        │ PC Web / H5  │
                        └──────┬───────┘
                               │ HTTP/WebSocket
                        ┌──────▼───────┐
                        │   FastAPI    │
                        │   API 服务   │
                        └──────┬───────┘
              ┌──────────────┼──────────────┐
       ┌──────▼──────┐ ┌────▼─────┐ ┌─────▼─────┐
       │  感知层      │ │  决策层   │ │  干预层    │
       │ Text+Audio  │ │ LLM风险   │ │ 分级预警   │
       │ +Image+Video│ │ 评估引擎  │ │ 监护人联动 │
       └──────┬──────┘ └────┬─────┘ └─────┬─────┘
              │             │             │
       ┌──────▼─────────────▼─────────────▼─────┐
       │               进化层                    │
       │  知识进化 ←── 数据源 ──→ ChromaDB向量库  │
       │  案例自动提取      关键词+模式双库       │
       └────────────────────────────────────────┘
```

### 2.3 四大核心模块

**感知层** —— 负责接收用户的文本、语音、图片、视频输入，进行预处理和多模态特征提取。

**决策层** —— 基于规则引擎 + LLM 大模型双通道进行风险评估，综合声学特征、语义特征、图像描述，给出最终风险评分。

**干预层** —— 根据风险等级触发对应的响应措施，从温和文字提醒到监护人强制通知，逐级升级。

**进化层** —— 定时从权威数据源抓取最新案例，自动提取特征、更新规则库，使系统能够应对最新诈骗手法。

## 第三部分：多模态感知能力（建议 6 分钟）

这是 SmartGuard 最核心的差异化能力。我们逐一介绍。

### 3.1 文本感知与风险分析

文本分析是最基础的入口。用户输入一段文字后，系统会做三层处理：

**第一层：关键词扫描**
内置超过 45 个高风险关键词，分为不同权重等级：
- **高危词**：转账、安全账户、验证码、检察院、拘捕令、资产审查
- **中危词**：投资、理财、利息、刷单、返利、注销
- **低危词**：快递、订单、退款、优惠

**第二层：组合模式匹配**
不只是看单个词，还会识别词之间的关系组合。比如以下 8 种模式：
1. 转账 + 验证码 = 极高危
2. 公检法 + 视频通话 = 高危
3. 投资 + 高回报 = 中危
4. 杀猪盘 + 情感 + 金钱 = 高危
5. AI语音 + 熟人语气 = 极高危
6. 刷单 + 垫付 = 中危
7. 贷款 + 先缴费 = 高危
8. 恐吓 + 限时 = 极高危

**第三层：LLM 语义深度分析**
当规则引擎判断为中高风险后，会调用通义千问大模型，从语义层面进一步分析对话意图，综合输出风险评级和详细分析理由。

### 3.2 语音感知——SenseVoice 集成

这是系统最亮点的功能之一。当用户发送语音消息时：

**步骤一：语音识别（ASR）**
调用阿里达摩院的 SenseVoiceSmall 模型，将语音转写为文字。SenseVoice 是目前业界领先的中文语音识别模型，支持情绪识别和音频事件检测。

**步骤二：声学特征提取**
使用 librosa 库提取六维声学特征：
- MFCC（梅尔频率倒谱系数）—— 反映音色和说话人特征
- 频谱质心 —— 反映音色明亮度
- 基频（F0）—— 反映音调高低
- 能量（RMS）—— 反映音量大小
- 频谱滚降点 —— 反映音色丰富度
- 语速（ZCR 过零率）—— 反映说话节奏

**步骤三：情感与异常检测**
- **情绪识别**：SenseVoice 内置 happy / sad / angry / neutral 四种情绪检测
- **音频事件检测**：识别 SPEECH / SILENCE / MUSIC / LAUGHTER / CRYING / APPLAUSE 等事件
- **声学异常评分**：当检测到语速异常加快、音调异常升高、能量异常波动时，触发风险加分

**步骤四：双通道风险融合**
将 ASR 转写的文本送入文本分析通道，将声学特征送入异常检测通道，两个通道的结果加权融合，得到最终风险评分。

### 3.3 图像感知

当用户发送图片时：

**图像描述**：调用 Qwen-VL-Max 多模态大模型，生成图片的详细文字描述。例如一张"公检法证件"图片，系统会识别出其中的文字内容、人物穿着、徽章样式等细节。

**目标检测**：集成 YOLO 目标检测模型，识别图片中的特定物体，如手机、银行卡、身份证、人民币等高风险物件。

**OCR 文字识别**：使用 PaddleOCR 提取图片中的文字内容，送入文本分析通道。

### 3.4 多模态融合

SmartGuard 支持同时传入文本 + 语音 + 图片的组合输入（如当前 API 中的 `/analyze/multimodal` 接口）。系统会对每种模态分别计算风险分，然后按权重融合：

```
最终风险分 = 文本权重 × 文本分 + 语音权重 × 语音分 + 图像权重 × 图像分
```

## 第四部分：智能决策引擎（建议 5 分钟）

### 4.1 规则引擎

规则引擎是基于人工专家知识构建的确定性判断逻辑，响应速度快、可解释性强。引擎内置：

- **45+ 风险关键词**：带权重系数和共现关系
- **8 种诈骗组合模式**：跨关键词的逻辑组合
- **12 种诈骗类型分类**：从冒充公检法到 AI 诈骗全覆盖
- **用户画像差异化策略**：老年人、未成年、财会人员有不同的权重配置

### 4.2 大模型决策

规则引擎给出初判后，中高风险内容会进入大模型决策通道。通义千问会从以下维度进行分析：

- **意图分析**：对方想让我做什么？
- **合理性判断**：这个要求合理吗？有没有不合逻辑的地方？
- **情感操控识别**：有没有使用恐惧、紧迫、爱情等情感操控手法？
- **身份核实建议**：建议用户通过什么渠道核实对方身份？

大模型的输出不是直接替代规则，而是作为"高级顾问"给出补充判断，最终由决策引擎加权综合。

### 4.3 ChromaDB 向量知识库

系统维护了一个诈骗案例知识库，基于 ChromaDB 构建。包含：
- **600+ 历史真实案例**：从公开渠道整理的典型诈骗案例
- **768 维向量索引**：每个案例通过嵌入模型转换为向量，支持语义相似度检索
- **关键词+模式双索引**：既支持精确关键词匹配，也支持语义相似检索

当用户输入一段对话时，系统会同时：
1. 在规则引擎中匹配关键词和模式
2. 在向量知识库中检索最相似的历史案例
3. 将相似案例作为 RAG（检索增强生成）的上下文提供给大模型

这样大模型不仅有自己的知识，还能结合知识库中的真实案例给出更准确的分析。

## 第五部分：分级干预响应（建议 4 分钟）

### 5.1 五级风险响应体系

SmartGuard 将风险分为 **5个等级**（0-4），每个等级有明确的响应动作：

| 等级 | 名称 | 触发条件 | 响应动作 |
|------|------|----------|----------|
| 0 | 安全 | 无风险信号 | 正常对话响应 |
| 1 | 低风险 | 1-2 个低危词 | 温和文字提醒 |
| 2 | 中风险 | 中危词组合 或 LLM 判定 | 弹窗警告 + APP 推送 |
| 3 | 高风险 | 高危词组合 | 强制阻断 + 预警记录 |
| 4 | 极高危 | 极高危词组合 | 监护人通知 + 建议报警 |

> **说明**：代码实现中共有0-4共5个风险等级，等级4为最高风险等级，会自动通知监护人。

### 5.2 监护人联动机制

当风险达到极高危等级时，系统会：
1. 向用户账户绑定的监护人发送预警通知
2. 记录完整对话日志供后续取证
3. 提供一键举报功能，用户可一键将对话提交到审核队列

### 5.3 预警管理后台

用户和监护人可以在前端查看：
- 所有收到的预警记录
- 每条预警的风险等级、触发原因、分析详情
- 预警的确认/处理状态

### 5.4 举报与案例入库

系统内置举报流程：用户举报 → 案例审核 → 特征提取 → 知识库更新。这形成了一个从用户反馈到系统进化的完整闭环。

## 第六部分：自适应进化系统（建议 5 分钟）

这是 SmartGuard 与传统反诈系统最本质的区别——**它能够自己学习**。

### 6.1 数据源体系

知识进化模块内置了对接多个权威数据源的抓取引擎：

| 数据源 | 类型 | 更新频率 |
|--------|------|----------|
| 公安部官方公告 | HTTP API | 每日 |
| 12321 举报中心 | HTTP API | 每日 |
| CCTV 新闻 RSS | RSS 订阅 | 每小时 |
| 权威媒体新闻 | 关键词爬取 | 每日 |

### 6.2 自动进化流程

整个进化流程是全自动的：

```
定时触发（每日凌晨 2:00）
    ↓
从各数据源抓取最新诈骗案例
    ↓
案例解析：提取诈骗类型、手法、关键词、目标人群
    ↓
案例验证：去重 + 格式校验
    ↓
标准化处理：归一化、清洗
    ↓
存入 SQLite 历史案例库
    ↓
生成向量，存入 ChromaDB
    ↓
更新规则引擎关键词权重
    ↓
进化统计记录
```

### 6.3 手动学习入口

除了自动进化，系统也支持管理员手动导入案例：
- 通过 API 提交案例文本
- 系统自动解析关键词和风险模式
- 审核后入库

### 6.4 进化效果评估

系统会记录每次进化的效果：
- 本次新增了多少案例
- 更新了哪些关键词权重
- 覆盖了多少种诈骗类型
- 向量库规模增长情况

## 第七部分：API 接口与集成能力（建议 4 分钟）

### 7.1 核心 API 一览

SmartGuard 提供了完整的 RESTful API，可以作为独立服务运行，也可以集成到其他系统中。

**对话接口**（最常用）：
```
POST /api/v1/chat
Body: { message, session_id, user_id, mode }
Mode 支持三种模式：
  - risk: 风险分析模式
  - chat: 智能助手模式
  - learn: 学习案例模式
```

**多模态分析接口**：
```
POST /api/v1/analyze/multimodal
支持同时上传 text + audio + image
一次请求返回：文本分析结果 + 语音转写 + 声学特征 + 图像描述 + 综合风险评分
```

**用户管理接口**：
```
GET/PUT  /api/v1/users/{user_id}/profile      # 用户画像
GET/POST /api/v1/users/{user_id}/guardians    # 监护人管理
GET      /api/v1/users/{user_id}/alerts       # 预警记录
```

**知识管理接口**：
```
GET /api/v1/knowledge/scam-types               # 12种诈骗类型列表
GET /api/v1/knowledge/cases                   # 相似案例检索
GET /api/v1/knowledge/statistics              # 知识库统计
```

**进化控制接口**：
```
POST /api/v1/evolution/evolve                 # 手动触发进化
POST /api/v1/evolution/manual-learn          # 手动导入案例
GET  /api/v1/evolution/stats                 # 进化统计
```

### 7.2 集成方式

系统支持两种部署模式：

**模式一：独立部署**
直接启动 API 服务，前端通过 HTTP 调用，适合作为独立的反诈服务使用。

**模式二：嵌入集成**
将 `AntiFraudAgent` 类作为模块嵌入到现有业务系统中，通过 Python API 调用其方法。

### 7.3 支持的诈骗类型

系统目前支持识别和分析以下 12 种主流诈骗类型：

1. 冒充公检法诈骗 —— 骗子冒充警察/检察官/法官
2. 投资理财诈骗 —— 高回报投资陷阱
3. 兼职刷单诈骗 —— 先垫付后返利的骗局
4. 虚假贷款诈骗 —— 贷款要先交费的陷阱
5. 杀猪盘诈骗 —— 网恋交友后引导投资
6. AI 语音合成诈骗 —— 用 AI 克隆熟人声音
7. 深度伪造诈骗 —— 用 AI 换脸视频诈骗
8. 虚假征信诈骗 —— 谎称征信有问题需要修复
9. 购物退款诈骗 —— 假借退款实施诈骗
10. 游戏交易诈骗 —— 游戏装备/账号交易陷阱
11. 追星诈骗 —— 粉丝群里的诈骗
12. 医保诈骗 —— 冒充医保局诈骗

SmartGuard 的核心价值在于：它不只是一个固定规则的反诈系统，而是一个**会学习、会进化、能感知多模态信号**的智能体。

从技术角度看，这套系统融合了：
- 大模型（通义千问）的语义理解能力
- 语音 AI（SenseVoice）的声学分析能力
- 视觉 AI（Qwen-VL）的图像理解能力
- RAG（ChromaDB）的知识检索能力
- 自动化爬虫的知识进化能力

从产品角度看，它真正解决了反诈领域的三个痛点：响应滞后、维度单一、缺乏进化。

未来我还计划继续扩展：

- 接入实时诈骗电话拦截
- 增加区块链存证功能
- 开发微信/钉钉机器人插件

## 附录：快速启动指南

### 环境要求
- Python 3.11
- Windows / Linux / macOS
- 8GB+ RAM（GPU 更佳）

### 启动步骤
```bash
# 1. 进入项目目录
cd agent

# 2. 配置环境变量（在 agent/.env 中设置 DASHSCOPE_API_KEY）

# 3. 启动服务
cd agent/src
python -m uvicorn src.api.main:app --host 0.0.0.0 --port 8000 --reload

# 4. 访问前端
# PC端: http://localhost:8000
# 移动端: http://localhost:8000/mobile
```

### 配置说明
| 环境变量 | 说明 | 默认值 |
|----------|------|--------|
| DASHSCOPE_API_KEY | 通义千问 API 密钥 | 必填 |
| QWEN_MODEL | 使用的模型 | qwen-turbo |
| DATABASE_PATH | 数据库路径 | ./data/anti_fraud.db |
| KNOWLEDGE_BASE_PATH | 知识库路径 | ./data/knowledge_base |
| LOCAL_MODEL_PATH | 本地模型路径 | 可选 |
