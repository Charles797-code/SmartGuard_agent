## 1. 系统概述

SmartGuard 是一款基于人工智能的多模态反诈智能助手，系统通过深度学习和大语言模型技术，实现对电信网络诈骗的智能识别、风险评估、实时预警和多方联动防护。

### 1.1 核心特性

| 特性 | 说明 |
|------|------|
| **多模态感知** | 支持文本、音频、图片三种输入方式 |
| **智能识别** | 基于LLM和规则引擎的诈骗识别 |
| **多级预警** | 5级风险评估体系，差异化响应 |
| **多方联动** | 支持监护人通知、举报等功能 |
| **持续进化** | 内置学习机制，不断优化识别能力 |
| **隐私保护** | 本地处理，数据不外传 |

## 3. 多模态感知能力

SmartGuard 支持同时处理文本、语音、图片三种输入，实现全方位的风险感知。

### 3.1 文本分析

**输入方式**：
- 直接输入文字内容
- 支持粘贴短信、聊天记录、网页内容等

**分析能力**：
- 语义理解：理解文字深层含义
- 关键词检测：识别诈骗关键词
- 上下文关联：结合会话历史分析
- 意图识别：判断用户目的

**示例测试**：
```
输入: "您好，我是公安局民警，您的身份证涉嫌一起洗钱案件，
      请立即将资金转入安全账户进行核查，否则将冻结您的账户"

预期结果: 风险等级 4 (高危诈骗)
```

### 3.2 语音识别 (Audio)

**输入方式**：
- 上传音频文件（MP3、WAV、M4A等格式）
- 支持实时语音消息

**分析能力**：
- 语音转文字（ASR）
- 情感分析（高兴/悲伤/愤怒/中性）
- 音频事件检测（音乐/笑声/哭声/静音）
- 语速语调分析

**处理流程**：
```
音频上传 → SenseVoice语音识别 → 情感分析 → 文本风险评估
```

**返回数据**：
```json
{
  "transcription": "骗子自称民警...",
  "emotion": "angry",
  "audio_events": ["SPEECH"],
  "risk_level": 4
}
```

### 3.3 图片识别 (Visual)

**输入方式**：
- 上传图片文件（截图、照片等）
- 支持聊天界面直接发送图片

**分析能力**：
- 图像语义理解（Qwen-VL）
- 文字识别OCR（PaddleOCR）
- 截图内容分析
- 二维码/链接检测

**返回数据**：
```json
{
  "image_description": "这是一张微信聊天截图，显示...",
  "detected_text": "请登录以下链接验证身份...",
  "suspicious_links": ["http://fake-bank.com"],
  "risk_level": 3
}
```

### 3.4 多模态融合

系统支持同时输入文本+音频+图片，综合分析：

```python
# API调用示例
POST /api/v1/analyze/multimodal

Form Data:
- text: "这是我收到的短信"
- audio: [文件]  # 可选
- image: [文件]  # 可选
- user_id: "user123"
```

**融合分析流程**：
```
     ┌─────────────┐
     │   文本输入   │──→ 文本分析
     └─────────────┘
     ┌─────────────┐
     │   音频输入   │──→ 语音识别 → 情感分析
     └─────────────┘
     ┌─────────────┐
     │   图片输入   │──→ 图像描述 → OCR识别
     └─────────────┘
            │
            ▼
     ┌─────────────┐
     │  融合分析   │──→ 综合风险评估
     └─────────────┘
```

## 4. 三大工作模式

SmartGuard 提供三种工作模式，用户可通过侧边栏一键切换。

### 4.1 风险分析模式 (Risk)

**图标**: 🔍

**功能**: 分析可疑内容是否为诈骗

**使用场景**：
- 收到可疑短信/电话内容
- 遇到疑似诈骗的情况
- 不确定某个信息是否可信

**交互示例**：
```
用户: 收到一条短信说我中了大奖，要我填写信息领奖
系统: ⚠️ 这很可能是一个中奖诈骗！
      
      【分析】
      这种"中奖"骗局有以下特征：
      • 要求先交手续费/税费
      • 奖品价值过高不合常理
      • 要求提供银行卡信息
      
      【建议】
      1. 不要点击任何链接
      2. 不要填写个人信息
      3. 直接删除短信
      4. 可以向12321举报
```

### 4.2 智能助手模式 (Chat)

**图标**: 💬

**功能**: 智能问答，了解反诈知识

**使用场景**：
- 询问某种诈骗类型的特征
- 了解最新的诈骗手法
- 获取防骗建议

**交互示例**：
```
用户: 什么是杀猪盘诈骗？
系统: 🛡️ 你好！让我来为你介绍杀猪盘诈骗：
      
      【什么是杀猪盘？】
      「杀猪盘」是一种网络交友诈骗，骗子通过婚恋网站、

      【典型特征】
      • 通过婚恋网站主动搭讪
      • 聊天几天就建立恋爱关系
      • 频繁提到投资、博彩、平台

      【真实案例】
      王女士被骗28万元...
```

## 5. 风险评估体系

### 5.1 五级风险评估

| 等级 | 名称 | 颜色 | 含义 | 响应措施 |
|------|------|------|------|----------|
| 0 | 安全 | 🟢 绿色 | 内容安全无风险 | 正常通过 |
| 1 | 低风险 | 🔵 蓝色 | 有轻微可疑之处 | 提示注意 |
| 2 | 中风险 | 🟡 黄色 | 存在一定风险 | 警告提示 |
| 3 | 高风险 | 🟠 橙色 | 很可能诈骗 | 强烈警告 |
| 4 | 极高危 | 🔴 红色 | 确认诈骗 | 阻断+通知监护人 |

> **说明**：系统采用 0-4 共5级风险体系，其中等级4为最高风险，会自动通知监护人。

### 5.2 评估维度

系统从多个维度综合评估风险：

| 维度 | 说明 | 权重 |
|------|------|------|
| **关键词匹配** | 诈骗关键词出现次数 | 高 |
| **语义分析** | LLM理解内容意图 | 高 |
| **上下文关联** | 结合历史会话分析 | 中 |
| **行为模式** | 用户交互行为分析 | 中 |
| **外部情报** | 知识库案例匹配 | 中 |

### 5.3 风险类型分类

系统支持识别以下诈骗类型：

| 类型 | 关键词/特征 |
|------|-------------|
| 冒充公检法 | 洗钱、犯罪、安全账户、警官证 |
| 投资理财诈骗 | 高收益、保本、内幕消息、平台 |
| 杀猪盘 | 网恋、投资、博彩、导师 |
| 刷单返利 | 点赞、刷单、返利、佣金 |
| 虚假贷款 | 无抵押、低利息、手续费 |
| AI语音诈骗 | 家人出事、声音合成、紧急 |
| 视频深度伪造 | 视频通话、AI换脸 |
| 虚假征信 | 征信修复、逾期、黑名单 |
| 购物退款 | 客服、退款、物流问题 |
| 游戏交易 | 装备、账号、充值 |
| 追星诈骗 | 粉丝群、打榜、签名 |
| 医保诈骗 | 医保卡、冻结、报销 |

## 17. 风险等级说明

### 17.1 等级0 - 安全 🟢

**定义**: 内容安全，无任何风险特征

**示例**:
- 日常问候
- 正常商务沟通
- 常规新闻资讯

**系统响应**: 正常显示，无需干预

### 17.2 等级1 - 低风险 🔵

**定义**: 有少量可疑特征，但整体风险较低

**示例**:
- 轻微夸大宣传
- 商业推销话术
- 模糊的投资建议

**系统响应**: 温和提示，建议留意

### 17.3 等级2 - 中风险 🟡

**定义**: 存在明显风险特征，可能是诈骗

**示例**:
- 提及安全账户
- 夸大收益承诺
- 要求提供敏感信息

**系统响应**: 弹窗警告 + 记录

### 17.4 等级3 - 高风险 🟠

**定义**: 高度疑似诈骗，需要警惕

**示例**:
- 典型诈骗话术
- 冒充官方机构
- 涉及账户异常

**系统响应**: 强制警告 + 阻断建议

### 17.5 等级4 - 极高危 🔴

**定义**: 基本确认是诈骗，必须立即处理

**示例**:
- 明确要求转账
- 威胁恐吓内容
- 涉及资金账户

**系统响应**: 全屏警告 + 监护人通知

> **说明**：等级4为系统最高风险等级，会自动通知监护人。
