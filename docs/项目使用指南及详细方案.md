## 1. 项目概述

### 1.1 项目背景

随着诈骗案件的高发，全民反诈已成为国家和社会的重要任务。本项目基于大语言模型（LLM）和多模态大模型（VLM），设计并开发了一个具备"感知-决策-干预-进化"能力的智能助手。

### 1.2 项目目标

- 提供多模态输入支持（文本、语音、图像、视频）
- 实现智能识别与决策引擎
- 提供实时干预与交互能力
- 构建自适应进化模块
- **支持邮件监控，自动检测邮箱中的可疑邮件**

##### 大语言模型（LLM）

### 1.5 技术亮点与创新

本项目在反欺诈智能助手领域具有多项技术创新和独特设计，主要体现在以下几个方面：

#### 1.5.1 多模态感知架构

本系统支持 **文本(Text)、音频(Audio)、图像(Image)** 三种模态的输入。各模态独立提取为文字后，拼接到 prompt 中交给 LLM 进行综合语义理解和风险判断。

**实际处理流程（`agent/src/api/main.py`）**：

```
用户上传 音频/图片/文本
    ↓
[AudioInputHandler] SenseVoice 语音转文字 → audio_text
[VisualInputHandler] VLM(Qwen-VL) 图像语义分析 → image_desc
    ↓
combined_text = text + "\n\n【语音转写】\n" + audio_text
combined_text += "\n\n【图片描述】\n" + image_desc
    ↓
decision_engine.assess_risk(combined_text)  ← 整体作为文本处理
    ↓
LLM → 风险判断
```

> **重要说明**：`agent/src/core/decision.py` 中的 `fuse_multimodal_risk` 方法以及 `agent/src/modules/recognizer/fusion.py` 中的 `MultimodalFusion` 类均为预留设计，**当前版本从未被调用**。多模态融合完全在 LLM 的语义理解层面完成，不存在独立的规则引擎级多模态融合模块。

---

##### 文本处理（`input_handler/text.py` → `TextInputHandler`）

```

    ↓
清洗文本（去空白、规范化引号、移除URL/电话号码/银行卡号）
    ↓
语言检测（基于中英文字符数量）
    ↓
实体提取（电话号码、金额、银行相关、机构名称）
    ↓
意图识别（咨询/投诉/转账/账号/个人信息）
    ↓
风险指标检测（金钱/紧迫/威胁/保密/权威/奖励关键词 + 特殊模式匹配）
```

支持的来源：`manual`（手动输入）、`chat`（聊天记录）、`sms`（短信）、`ocr`（图片文字识别）。

内置风险信号词库（6类）和意图模式（5类），自动标注 `转账`、`验证码`、`安全账户`、`密码`、`投资`、`高收益`、`保本`、`无抵押`、`刷单` 等高风险关键词。

##### 音频处理（`input_handler/audio.py` → `AudioInputHandler`）

```
音频字节
    ↓

    ├── ASR 语音识别（支持中文/英文，自动反向文本标准化 ITN）
    ├── 情感识别（happy/sad/angry/neutral）

    ↓

    ├── 关键词匹配（8类高危词：转账汇款/验证码泄露/冒充身份/威胁恐吓等）
    └── 组合模式检测（急迫+转账/威胁+安全账户/保密+屏幕共享）

情感信号融合（模型输出 + 关键词增强）
```

| 语速 | > 6 字/秒 | < 2 字/秒 | 2.0 / 1.0 |
| 停顿比例 | > 40% | - | 1.5 |

##### 决策引擎（`agent/src/core/decision.py` → `RiskDecisionEngine`）

决策引擎是纯文本规则引擎，输入为 `_preprocess_input` 拼接后的 `multimodal_description`（含文本、音频转写、图片描述），**不区分模态来源**。

**评估流程（`assess_risk`）**：

```
输入文本（多模态拼接字符串）
    ↓
1. 关键词检测（_detect_keywords）
   遍历 KEYWORD_WEIGHTS 字典，命中则记录 (keyword, weight)
    ↓
2. 关键词得分计算（_calculate_keyword_score）
   score = Σ(weight) + 0.5 × (命中数 - 1)  ← 多重触发奖励
   上限 10.0 分
    ↓
3. 诈骗模式匹配（_match_scam_patterns）
   检查 SCAM_PATTERNS 中每种诈骗类型的必需关键词：
   - 全部命中 → 直接匹配
   - 命中 ≥ 2 → 部分匹配
   pattern_score = 匹配数 × 1.5
    ↓
4. 上下文增强（_analyze_context_boost）
   最近高风险对话 +1.0 / 风险升级趋势 +1.5 / 涉钱+紧迫 +2.0
    ↓
5. 基础分 = 关键词得分 + 模式得分 + 上下文增强
    ↓
6. 用户画像调整（_apply_profile_adjustment）
   根据年龄群体/职业叠加额外分数：
   - 老年人/未成年人：基础分+1，涉钱词权重翻倍
   - 财会人员：交易词额外+2.0
    ↓
7. 风险类型判定（_determine_risk_type）
   优先用模式匹配结果；无模式时按关键词推断
    ↓
8. 分数转等级（_score_to_level）
   [0, 2.0) → 0 安全 / [2.0, 4.0) → 1 关注 / [4.0, 6.0) → 2 警告 /
   [6.0, 8.0) → 3 危险 / [8.0, +∞) → 4 紧急
    ↓
9. 生成分析文本 + 建议 + 警告信息 + 推荐行动
    ↓
返回 RiskAssessment
```

**关键词权重库**（KEYWORD_WEIGHTS，45+ 词条，分 10 类诈骗）：

| 诈骗类型 | 代表关键词 | 权重范围 |
|---------|-----------|---------|
| 冒充公检法 | 安全账户、拘捕令、洗钱、验证码 | 1.5 ~ 3.0 |
| 投资理财 | 稳赚不赔、保本、导师带单、博彩 | 1.5 ~ 3.0 |
| 兼职刷单 | 刷单、点赞、日结、佣金 | 1.0 ~ 2.5 |
| 贷款诈骗 | 无抵押、快速放款、解冻、手续费 | 1.0 ~ 2.0 |
| 杀猪盘 | 亲爱的、博彩平台、下注 | 1.0 ~ 2.5 |
| AI语音诈骗 | 绑架、出事、急需用钱、汇款 | 2.0 ~ 3.0 |
| 征信诈骗 | 征信、逾期、洗白、修复 | 1.5 ~ 2.5 |
| 退款诈骗 | 双倍赔偿、退款、备用金 | 1.5 ~ 2.0 |

**诈骗模式库**（SCAM_PATTERNS，8 种组合模式）：每种模式定义必需关键词 + 附加分 + 上下文词表。例如"冒充公检法"要求命中 `["公安", "民警", "警官", "洗钱", "涉嫌"]` 中至少 2 个。

**用户画像调整**（PROFILE_ADJUSTMENTS）：对老年人/未成年人/财会人员等高风险群体，在基础分上叠加额外权重，对涉钱操作更严格。

**置信度计算**：关键词贡献 40%、模式匹配贡献 40%、上下文增强贡献 20%，有模式匹配时总分乘以 1.2 倍。

**调用位置**（`agent/src/core/agent.py`）：
- `_assess_risk`（line 568）：无 LLM 时的唯一判断路径
- `_llm_full_analysis`（line 677）：LLM 分析前预检关键词，供提示词参考
- `_llm_full_analysis`（line 760）：LLM 降级时的兜底路径

> `fuse_multimodal_risk` 方法为预留设计，当前未接入。

#### 1.5.9 自动进化学习系统

系统具备从用户举报和案例中自动学习的能力，不断扩展知识库。进化模块分为 **4 个文件**，职责分层：

| 文件 | 职责 | 核心类 |
|------|------|--------|
| `evolution/learner.py` | NLP 知识提取 | `KnowledgeLearner`、`ChineseTextProcessor`、`PatternExtractor` |
| `evolution/updater.py` | 外部知识更新 | `KnowledgeUpdater`、`DataSourceRegistry`、`ScheduledUpdater` |
| `evolution/report_integration.py` | 举报融合 | `ReportFusionEngine`、`ReportQualityEvaluator` |
| `services/evolution_service.py` | 核心编排层 | `EvolutionService`、`KnowledgeManager` |

##### 两条知识输入通道

**通道一：用户举报 → 进化（主路径）**

```
用户提交举报
    ↓
ReportService.get_reports_for_evolution()
    ↓
ReportQualityEvaluator 四维评估
  - 完整性：字数、关键字段、金额、证据
  - 可靠性：来源、举报状态、内容一致性
  - 新颖性：新型手法检测、与已知模式重叠度
  - 多样性：句式、关键词数量、实体类型
    ↓
质量过滤 (质量分 >= 0.4 才进入学习流程)
    ↓
ReportFusionEngine 去重融合
  - 相似案例合并（文本相似度 > 85%）
  - 关键词聚类、模式聚类
    ↓
KnowledgeLearner 提取关键词/模式
  - 中文分词 + 二元组/三元组提取
  - TF-IDF + 区分度 + 上下文权重计算
  - 槽位抽象化模式提取（<金额>、<身份>等）
    ↓
KnowledgeManager 持久化到数据库
    ↓
RiskDetectionEnhancer 将新知识注入风险检测引擎
```

**通道二：外部数据源 → 定时更新**

```
ScheduledUpdater 定时触发
    ↓
DataSourceRegistry 管理多个外部源
  - 公安部公开数据（API）
  - 12321举报中心（Web抓取）
  - CCTV新闻 RSS
    ↓
DataFetcher 异步抓取（aiohttp）
    ↓
CaseParser → CaseValidator → KnowledgeUpdater
```

##### 核心状态机

```
IDLE（空闲）
    ↓ [积累 >= 10 个未学习案例]
LEARNING（学习）
    ↓ [提取关键词和模式]
EVALUATING（评估）
    ↓ [计算准确率提升]
OPTIMIZING（优化）
    ↓ [将新知识注入检测引擎]
IDLE（空闲）
```

触发方式：自动进化（每分钟检查）、定时举报同步（每5分钟）、手动触发 `trigger_evolution()`

##### 知识持久化

| 数据表 | 字段 | 说明 |
|--------|------|------|
| `evolution_keywords` | keyword, scam_type, weight, frequency, is_verified | 关键词独立存储 |
| `evolution_patterns` | pattern_id, pattern_text, scam_type, confidence | 模式库 |
| `evolution_records` | 完整案例记录，含分析/回复内容 | 学习记录 |

支持知识快照（JSON + 校验和），可导出/导入/回滚。

##### 关键设计点

- **质量门槛**：质量分 >= 0.4 才学习，防止噪声数据污染
- **增量去重**：相似度 > 85% 的案例合并
- **渐进权重**：新知识权重由低到高，随使用频率逐步提升
- **双向反馈**：学到的知识实时注入检测引擎，检测增强后又反馈新案例

**代码位置**：`agent/src/modules/evolution/`、`agent/src/services/evolution_service.py`

### 6.2 核心模块设计

#### 6.2.1 Agent核心模块

```python
class AntiFraudAgent:
    """多模态反诈智能体"""
    
    def __init__(self, user_id, llm_client, knowledge_base_path, ...):
        self.user_id = user_id
        self.llm_client = llm_client
        self.memory = UserMemory(user_id)
        self.knowledge_retriever = KnowledgeRetriever(knowledge_base_path)
        self.alert_manager = AlertManager()
        self.guardian_notifier = GuardianNotifier()
        
    async def process(self, input_data: AgentInput) -> AgentOutput:
        # 1. 感知：多模态输入处理
        perceived = await self._perceive(input_data)
        
        # 2. 决策：风险评估
        decision = await self._decide(perceived)
        
        # 3. 干预：执行预警
        await self._intervene(decision)
        
        # 4. 进化：学习新知识
        await self._evolve(decision)
        
        return decision
```

#### 6.2.2 意图识别模块

```python
class IntentRecognizer:
    """意图识别器"""
    
    async def recognize(self, text: str, context: Dict) -> IntentResult:
        # 使用Prompt Engineering进行意图识别
        prompt = self._build_intent_prompt(text, context)
        result = await self.llm_client.chat([{"role": "user", "content": prompt}])
        return self._parse_intent_result(result)
```

#### 6.2.3 风险评估模块

```python
class RiskEvaluator:
    """风险评估器"""
    
    async def evaluate(self, input_data, user_profile) -> RiskAssessment:
        # 1. 关键词检测
        keyword_level = self._keyword_check(input_data.text)
        
        # 2. 案例匹配
        similar_cases = await self._match_cases(input_data.text)
        
        # 3. LLM分析
        llm_analysis = await self._llm_analyze(input_data, user_profile)
        
        # 4. 综合评分
        final_score = self._calculate_score(keyword_level, similar_cases, llm_analysis)
        
        return RiskAssessment(
            risk_level=final_score.level,
            risk_type=final_score.type,
            confidence=final_score.confidence,
            analysis=final_score.analysis
        )
```
